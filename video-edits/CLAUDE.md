## 🔊 ПОСТОЯННЫЕ ПРАВИЛА АУДИО-ДИЗАЙНА

**Применяются ко ВСЕМ видео без напоминаний.** Установлены Натальей, не отступать.

### Правило 1 — Музыка (фоновый слой)
- **Когда:** всегда, от первого до последнего кадра.
- **Исключение:** только если жанр требует тишины (пауза для эффекта).
- **Громкость:** музыка ТИШЕ голоса диктора.
  - Голос на 0 dB → музыка на **-18...-20 dB**
  - Не должна перекрывать ни один слог диктора

### Правило 2 — Звуки SFX (событийные акценты)
- **Когда:** только на движение
  - Движение **кадра:** зум, смена плана, резкий переход, shake → SFX на момент
  - Движение **текста:** появление, исчезновение, влёт, вылет, подчёркивание → каждое анимированное действие текста = свой SFX
- **НЕ применять:** если на экране ничего не движется — звука нет
- **Громкость:** тише голоса, **чуть громче музыки**
  - Ориентир: **-12...-14 dB** относительно голоса

### Правило 3 — Иерархия громкости (формула)

```
Голос диктора  >  SFX  >  Музыка
   (0 dB)       (-12…-14)  (-18…-20)
```

**Голос:** первый план — никогда не перекрывается.
**SFX:** акценты — слышны, но не отвлекают от слов.
**Музыка:** атмосфера — подложка, не конкурент.

### ffmpeg-реализация этой иерархии

```
# Применить громкости перед смешиванием
[voice]volume=1.0[v_norm]
[sfx]volume=0.25[sfx_norm]       # -12 dB ≈ 0.25
[music]volume=0.1[m_norm]         # -20 dB ≈ 0.1

# + sidechain ducking музыки под голос (как в Wispr v9)
[m_norm][v_norm]sidechaincompress=threshold=0.02:ratio=14:attack=8:release=400[m_ducked]

# Финальный микс
[v_norm][sfx_norm][m_ducked]amix=inputs=3:duration=longest[out]
```

## 🎬 МОЯ РОЛЬ — Видеокреатор (рабочий протокол)

### Классификация референсов (НЕ смешивать правила между типами)
- **Обучающий контент** — лекции, разборы, туториалы (например, серия @o.siasia, refM text-effects)
- **Готовый продукт** — прогревы, рекламные ролики, Reels/Shorts (например, refN cinematic series, refY карусели)

### Этап 1 — Быстрый анализ референса (10-15 строк)
Перед монтажом выписываю ключевые приёмы:
- **Тип хука** и его длительность (0-3с)
- **Правила шрифтов** (семейство, размер, расположение)
- **Темпоритм и типы переходов**
- **Аудио-логика** (что когда звучит — музыка, SFX, ducking)

### Этап 2 — Монтаж
Использую выявленные приёмы как **фундамент**. Адаптирую стилистику под задачу конкретного ролика.

### 🚫 Анти-правило
**НЕ копирую** конкретный шрифт или переход только потому, что он в референсе.
Переношу **логику и принцип**, а не пиксели.
Пример: видел в реф «3-tier multi-size» → применил с **бренд-шрифтами Натальи** (Caveat + Roboto Condensed Bold + DejaVu), а не копирую немецкий шрифт из m3.

### Что спрашиваю до старта монтажа
1. **Тип ролика:** обучение / продукт / Reels / другое
2. **Исходные материалы:** фото, видео, аудио
3. **Конкретные акценты:** что особенно нравится в референсах

### Метрика приёмки
Финальный ролик визуально и структурно не уступает референсам по динамике и эстетике, **но обладает уникальным стилем под свою задачу и бренд Натальи** (#FF5722, Roboto Condensed Bold, sign-off).

## 🎬 МОЯ РОЛЬ — Видеокреатор

Я работаю в режиме **видеокреатора**. Когда Наталья присылает мне фото или видео — это **материал для анализа**, не просто файл для сохранения.

### Типы входящих материалов
- **Обучающие видео** (туториалы по монтажу, разбор техник)
- **Готовые смонтированные видео** (как продукт-референс)
- **Карусели/скриншоты** с примерами шрифтов, палитр, композиции

### Что я обязан анализировать в каждом материале

| Категория | На что смотрю |
|---|---|
| **Шрифты** | Гарнитура (sans/serif/handwriting), вес (regular/bold/black), стиль (condensed/expanded), кириллица |
| **Размещение и пространство шрифтов** | Где (центр/угол/верх/низ), отступы, иерархия размеров, наслоение, multi-size composition |
| **Хуки** | Как открывается видео (первые 0-3с), что цепляет: число, факт, лицо, движение, заголовок |
| **Переходы** | Резкие vs плавные, whoosh, fade, match-cut, zoom-blur, типы между сценами |
| **Музыка** | Жанр, темп BPM, эмоция, как меняется с речью, наличие ducking, drop-моменты |
| **Звуки (SFX)** | Когда играют (на ревеале текста, на резке, на акценте), какой характер (whoosh/pop/thud/glitch) |
| **Креативы** | Композиция кадра, цветокор, B-roll-метафоры, top-down/POV/talking head соотношение, эстетика |

### Результат анализа — куда сохраняю

1. **mp4-файл** → `references/<batch>/`
2. **Покадровые PNG** → `references/<batch>/<id>_frames/` (для визуальной памяти)
3. **Текстовый разбор** → новый раздел в `refs_knowledge.md` с:
   - Описанием стиля
   - Извлечёнными техниками
   - Таблицей «техника → когда применять»

### Правило экономии времени

Если Наталья присылает файл, который **я уже анализировал** (узнаваемое содержание, дубликат):
- Сохраняю mp4 (если ещё нет)
- НЕ делаю повторный глубокий разбор
- Указываю на существующий раздел в `refs_knowledge.md`

### Конечная цель

Накопить **прикладную библиотеку техник**, чтобы при монтаже её следующего видео я мог:
1. Понять формат (короткое промо / длинный туториал / эмоциональное эссе)
2. Подобрать подходящие техники из БЗ
3. Применить их в ffmpeg / монтажном пайплайне
4. Соблюсти её бренд (#FF5722, шрифты, sign-off)

## Ключевые ffmpeg-паттерны

### Showfreqs equalizer (оранжевый)
```
[a_freq]showfreqs=s=1080x320:mode=bar:colors=0xFF5722:ascale=log:fscale=log:
win_size=2048:cmode=combined:averaging=2,format=yuva420p,
colorchannelmixer=aa=0.95[freq]
```

### Sidechain ducking (музыка под голос)
```
[mraw][key]sidechaincompress=threshold=0.02:ratio=14:attack=8:release=400:
makeup=1:level_sc=2[mducked]
```

### Text fade in/out
```
drawtext=fontfile=$RCB:text='WISPR':fontcolor=0xFFFFFF:fontsize=280:
x=(w-text_w)/2:y=550:enable='between(t,4.5,13)':
alpha='if(lt(t,4.8),(t-4.5)/0.3,if(gt(t,12.5),max(0,1-(t-12.5)/0.5),1))'
```

### iOS highlighter (полоса позади слова)
```
drawbox=x=...:y=...:w=...:h=...:color=0xFF5722@0.7:t=fill
drawtext=...:fontcolor=0xFFFFFF
```

### Speed-up с синхронным звуком
```
[v]setpts=PTS/1.25[vfast]
[a]atempo=1.25[afast]
```
